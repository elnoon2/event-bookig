# 🚀 خطوات نشر المشروع على Railway — بالتفصيل وبالراحة

> اتبع الخطوات بالترتيب من فوق لتحت. متعملش خطوة قبل اللي قبلها.
> كل خطوة فيها مربع `[ ]` — حطّ علامة ✅ لما تخلّصها عشان متتلغبطش.

الكود اترفع على GitHub خلاص ✅. باقي بس إعدادات Railway و Azure.

---

## 🟦 الجزء الأول: إنشاء المشروع على Railway

### الخطوة 1 — افتح Railway وابدأ مشروع
- [ ] ادخل على https://railway.app وسجّل دخول.
- [ ] اضغط **New Project** (زرار فوق على اليمين).
- [ ] اختار **Deploy from GitHub repo**.
- [ ] اختار الريبو: **`elnoon2/event-bookig`**.
- [ ] Railway هيبدأ يبني المشروع لوحده. سيبه يشتغل دلوقتي وكمّل الخطوات اللي تحت.

> ملاحظة: أول build ممكن ياخد من 5 لـ 10 دقايق. عادي.

---

## 🟦 الجزء الثاني: إضافة قاعدة البيانات (مهم جداً)

> من غير الخطوة دي، أي طالب يسجّل ممكن تضيع بياناته لما السيرفر يعيد التشغيل.

### الخطوة 2 — ضيف MySQL
- [ ] وانت جوه المشروع، اضغط **New** (أو **+ Create**).
- [ ] اختار **Database**.
- [ ] اختار **Add MySQL**.
- [ ] استنى شوية لحد ما الداتابيز تبقى جاهزة (هتلاقي بوكس اسمه MySQL ظهر في المشروع).

## 🟦 الجزء الثالث: ضبط الإعدادات (Variables)

> دي أهم خطوة. لازم تكون على **سيرفس التطبيق** (اللي اسمه event-bookig)، **مش** على بوكس الـ MySQL.

### الخطوة 3 — افتح مكان الإعدادات
- [ ] اضغط على بوكس التطبيق (مش MySQL).
- [ ] افتح تبويب **Variables**.

### الخطوة 4 — ضيف المتغيرات دي واحد واحد
لكل واحد: اضغط **New Variable** → اكتب الاسم في خانة، والقيمة في الخانة التانية → **Add**.

انسخهم بالظبط زي ما هما:

```
SPRING_PROFILES_ACTIVE
```
القيمة:
```
prod
```

---
```
SPRING_DATASOURCE_URL
```
القيمة:
```
jdbc:mysql://${{MySQL.MYSQLHOST}}:${{MySQL.MYSQLPORT}}/${{MySQL.MYSQLDATABASE}}?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
```

---
```
SPRING_DATASOURCE_USERNAME
```
القيمة:
```
${{MySQL.MYSQLUSER}}
```

---
```
SPRING_DATASOURCE_PASSWORD
```
القيمة:
```
${{MySQL.MYSQLPASSWORD}}
```

---
```
MICROSOFT_CLIENT_ID
```
القيمة:
```
a3706e9d-46d0-4640-bfe2-ac84f5cc821c
```

---
```
MICROSOFT_CLIENT_SECRET
```
القيمة: **(انسخها من ملف `microsoft-env.bat` السطر اللي فيه MICROSOFT_CLIENT_SECRET)**

---
```
MICROSOFT_TENANT_ID
```
القيمة:
```
common
```

---
```
APP_ADMIN_EMAIL
```
القيمة: **(الإيميل اللي عايز تدخل بيه لوحة الأدمن)**

---
```
APP_ADMIN_PASSWORD
```
القيمة: **(باسورد قوي للأدمن — اكتبه واحفظه عندك)**

---
```
QR_SIGNING_SECRET
```
القيمة: **(أي كلام عشوائي طويل، مثلاً: badya-qr-2026-x9f3k2m8z)**

## 🟦 الجزء الرابع: إنشاء اللينك (الدومين)

### الخطوة 5 — اعمل دومين للمشروع
- [ ] وانت على سيرفس التطبيق، افتح تبويب **Settings**.
- [ ] انزل لقسم **Networking**.
- [ ] اضغط **Generate Domain**.
- [ ] هيطلعلك لينك شكله كده: `https://event-bookig-production.up.railway.app`
- [ ] **انسخ اللينك ده واحفظه** — هنستخدمه في الخطوات الجاية. 👇

> ✏️ اللينك بتاعك هو: ____________________________________________

### الخطوة 6 — ضيف 3 متغيرات كمان فيهم اللينك
ارجع لتبويب **Variables** وضيف التلاتة دول (**بدّل** `اللينك_بتاعك` باللينك اللي نسخته):

القيمة:
```
اللينك_بتاعك/api/auth/microsoft/callback
```
مثال: `https://event-bookig-production.up.railway.app/api/auth/microsoft/callback`

## 🟦 الجزء الخامس: ضبط Microsoft في Azure (الخطوة اللي بينساها الكل)

> ⚠️ من غير الخطوة دي **مفيش طالب هيقدر يسجّل** — هتظهر رسالة خطأ redirect.

### الخطوة 7 — ضيف رابط الـ callback في Azure
- [ ] ادخل على https://portal.azure.com
- [ ] دوّر فوق على **Microsoft Entra ID** وادخلها.
- [ ] من القايمة الجنب اختار **App registrations**.
- [ ] افتح التطبيق بتاعك (اللي الـ Client ID بتاعه `a3706e9d...`).
- [ ] من القايمة الجنب اختار **Authentication**.
- [ ] تحت **Platform configurations**:
  - لو لاقيت **Web** موجود: اضغط **Add URI**.
  - لو مش موجود: اضغط **Add a platform → Web**.
- [ ] اكتب الرابط ده (بلينكك):
  ```
  اللينك_بتاعك/api/auth/microsoft/callback
  ```
- [ ] اضغط **Save** (فوق).

## 🟦 الجزء السادس: التأكد إن كل حاجة شغّالة

### الخطوة 8 — جرّب الموقع
- [ ] افتح **اللينك بتاعك** في المتصفح → المفروض الصفحة الرئيسية تفتح وتلاقي الإيفنتات.
- [ ] اضغط **Sign in with Microsoft** وجرّب تسجّل بحساب → المفروض يدخل عادي.
- [ ] جرّب تحجز إيفنت → المفروض يتحجز ويطلعلك QR.

### الخطوة 9 — جرّب لوحة الأدمن
- [ ] افتح: `اللينك_بتاعك/admin.html`
- [ ] سجّل دخول بـ `APP_ADMIN_EMAIL` و `APP_ADMIN_PASSWORD` اللي حطيتهم.
- [ ] المفروض تقدر تضيف/تعدّل الإيفنتات وتشوف التسجيلات.
